4. br 태그 (=break)

5. a 태그 (=ancor)

(*attribute)

++원하는 곳으로 이동시킬때 (내부>외부, 외부>내부, 내부>내부...등등)
==> 이동 시킬때 이동하고자 하는 곳의 지점을 설정 <h4 id="home">(예시)
	<a href="#home"> --> 문법적으로 #(해시태그)를 붙이고 써야한다.

9. Table Tags (요새 추세는 아니지만 기본으로 알고 있어야 한다.)
테이블 표를 만들기 위한 태그 (세트로 같이 쓰인다. 보통)
>>사용하기전 행과 열을 구성 (행 : 가로줄 row, 열 : 세로줄 coulm)

- table (표를 명시) =>선을 보고 싶을때 border="1(px)" 속성을 주면 된다.
- tr (몇개의 행을 만들지 정의) - table row
- th (생성된 열안에서 강조하고자 할때) -대표문구 역할 table heading
- td (생성된 행 안에 몇개의 열을 만들지) - table data
-----------------------------------------------------------------------------------
《테이블 태그 내에 구성》 - 웹 표준 정식 방식이다.
- thead - 머리
- tbody - 몸통
- tfoot - 다리  (**해당 부분이 없을시 억지로 쓰지 않아도 된다.)

- colgroup (테이블 안 열을 선택그룹으로 나뉘어서 사용) 색상을 넣거나 그럴때
- col  - 콜그룹 내에 몇번째 열을 지칭해서 역할을 수행할 것인지
- caption - 테이블 전체를 통틀어 전용 타이틀 목록을 지정하고자 할 때
- colspan - 테이블 내에 열을 병합하고자 할때 (셀 합치기)
- rowspan - 테이블 내에 행을 병합하고자 할때

예시)
table>(tr>th+td*2)*4 이런식으로 수식
행 10 열 6
table>(thead>tr>th)+tbody>(tr>td*6)*9

14.공간분할태그 :
-인라인 과 블록 요소를 가지고 있다.
- block : 태생부투 욕심이 많은 욕심쟁이 1개의 행에 본인만 들어온다 (1줄을 차지)
	    굳이 너비 (width)값을 세팅하지 않아도 됨!
		>자연적인 text-align: center; (가운데 정렬) 이 가능하다

- inline : 태생부터 욕심이 없는 이타주의 성격 나의 너비값(px) 만큼만 차지(1줄에 여러개 가능)
		> Auto width 자기 너비만큼 딱맞는 공간을 차지하기 때문에 가운정렬이 안된다.

>> 두개의 공간분할 태그의 가장 큰 차이점
1) 수평 정렬 방식이 상이함
2) margin, padding 적용방식이 상이함
> block 태그는 상/하/좌/우 모두 적용가능
> inline 태그는 특별한 경우가 아니라면, 좌우측만 적용가능
 	(**특별한 경우는 부모요소에 flex 가 적용되어 있는 경우)
