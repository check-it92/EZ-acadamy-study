static(잘 안씀) > 정적인 // 움직임에 대한 변수가 존재하지 않는다.
			코딩을 작성한 순서대로 출력이 된다. 어떠한 태그요소를 작성하던 간 기본적인 default 속성!!
			/* 다른 상태로 변환 했다가 원상복구 초기 값을 되찾고 싶을 때 static를 쓴다. */

relative : 상대적인 // 자식요소 중에 position absolute(절대값)가 있는 경우, 부모요소에 relative 줘야한다.
				그러면 자식의 절대값이 최상위 부모값이 아닌 본인보다 1단계 상위의 부모값에 적용이 된다.
				만일, 상대적인 좌표값이라 기준점(static)이 있다면 relative 자체로 써도 된다.
				(절대값 없이 써도 된다.)
				**대신 위치값을 설정해주면 된다. 
				position: relative;
          				top: 28px;
          				left: 48px;

absolute : 절대적인 // 계속 상대적으로 계산을 한다면 중첩될 수록 복잡해지고 기준이 달라진다.
				때문에 최상위 부모요소를 기점으로 절대값을 고정해놓고 기준으로 움직이게 만든다.
				(부모의 명령만 듣고) 절대적인 공간을 가지게 되니 다른 공간 고정되어 있는걸로 인식이 된다.
				만약 부모 요소 중 누구도 relative 속성을 가지고 있지 않은 경우에는 최상위 루트태그
				중심으로 좌표값 생성 But, 부모값에 relative가 있다면 본인보다 1단계 상위 부모값에 적용
				만일, 최상위 부모부터 최하위 자식까지 절대값을 적용시켜도 아무문제는 없다.
				굳이 부모에게 relative를 안줘도 되는 경우도 있다.


fixed : 고정된 // 예시) 사이드바에 카테고리(날개배너)들로 활용 스크롤을 내려도 전환이 쉽도록
				고정된 형태로 정보를 제공

sticky : 끈적한 // sticker // 주로 상단 배너, 카테고리에 사용 우리가 스크롤을 내릴때 내림에 따라
				    상단에 붙어 따라온다.
				    **대신, 부모가 가지고 있는 공간에 따라 적용이 된다.
									(특정좌표)에 따라 스티커 처럼 작동이 된다.

-->> z-index: 0; (기준점) 태그가 가지고 있는 의미는 3차원의 의미를 지니고 있다. ( X, Y, Z)축
                  그렇기 때문에 우리가 보고있는 화면 앞으로 나오려 하는 위치 속성을 가지고 있다.
		 평상시에 우리가 값을 입력하지 않으면 0의 값을 가지고 있는 것이다.
		  **(양수로 설정할 경우 앞으로 나오고 // 음수로 설정할 경우 뒤로 들어간다.)

--->> overflow: scroll // hidden (화면 비율에 따라 적용해서 쓴다.)
		부모요소의 너비를 초과하였을 때.
-hidden : 설정된 공간안에 해당태그가 튀어나오지 않도록 안에 감추는 역할 해당 공간만큼만 보인다.

-scroll : 설정된 공간안에 해당태그가 튀어나오지 않도록 하면서 그 전체 내용을 스크롤을 통해 보여질수 있게끔해주는 역할 

-text-overflow: ellipsis 설정된 공간안에 해당 텍스트가 튀어나오지 않도록 말줄임표로 수정해주는 요소
