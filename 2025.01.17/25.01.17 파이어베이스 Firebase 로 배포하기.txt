1. 파이어베이스 Firebase 로 배포하기

해당 비주얼 소스코드에서 진행된다. 컨트롤 쉬프트 백틱 키
node

- node.js 에서 작동한다.
> 로컬 컴퓨터에 설치가 되어있어야 한다.
> cmd 에서 node -v 커맨드 입력
> firebase 에 회원가입 후 로그인 후 > 프로젝트 생성
> 사이드메뉴 > 빌드 > 호스팅(hosting) > 시작하기

2. 파이어베이스 CLI설치는 최초 1번만 하면 된다.
**1. 예외사항
- 컴퓨터 시스템 설정 오류 > 파이어베이스가 설치되었음에도
		파이어베이스 명령어를 인신하지 못하는 경우

시스템 환경변수에 가서 > 윈도우키 + r > sysdm.cpa || sysdm.cpl ,3 커맨드입력하기
> 환경변수 선택 후 > path > 편집 > 새로만들기  > 찾아보기 클릭
> 유저컴퓨터/npappdata/Roaming/npm

** 2. 안될 시 보안 사항 체크하기
- 좌측하단,돋보기 > powershell '관리자모드'로 접속 > 현재 보안정책 확인 Get-ExecutionPolicy 입력
> 만약, Resrricted : 보안정책이 엄격하게 제한 되어있음
- 보안정책 변경 : Set-ExecutionPolicy RemoteSigned 입력 (직접제어한다는 뜻)
> 예[y]를 입력하기

- 시스템 환경변수 혹은 powershell 보안정책 변경 된 후 vscode를 한번 재부팅하기
>>완료!!


3. 프로젝트 초기화
최초 1회 로그인 하기 firebase login 커맨드 입력

firebase init 입력하고 방향키로 호스팅 스페이스바 (선택) 후 엔터
그 후 public 로 하고 싱글페이지 앱 // 깃허브 여부 선택하기 (과제특성상 n을 선택)

4. 마지막 배포단계
firebase deploy 하면 완료

5. 재배포
배포후에 오류사항을 발견했다면
public 폴더 안 데이터들을 다 지우고
원본 데이터에서 오류사항을 해결하고

fibaseinit 부터 초기화부터 다시 시작하기
만약 캐시값이 쌓여 있다면 오류된 상태의 파일을 지워야 해서
배포된 사이트의 화면에서
ctrl + shift + r 키를 눌러 리셋을 하고 올리면 된다.
