자바스크립트에서 **이벤트란? 무엇이냐?

- 브라우저를 통해서 특정 웹페이지 도착
- 사용자가 하는 행위 모든것
- 페이지를 확인 // 버튼 클릭 // 스크롤 이동

- addEventListener() 함수를 많이  활용한다.
=> 예전에는 event handler >> onclick 이런식으로 써서 활용을 했다.
왜? 요즘에는 콜백함수라 던지 여러가지 중첩적으로 활용하기 위해서 이 함수를 활용하기 시작
- 이벤트 설정 > 이벤트 실행 > 로컬 컴퓨터 > 메모리 공간 > 이벤트 객체 생성
>> 이벤트 객체를 참조변수 값으로 찾아올 수 있다.
- 이벤트 객체 > prevenDefault() 유일한 메서드 함수

1. ** 이벤트 속성종류

1) 문서(document) 로딩 이벤트 : 'on' 이라는 전치사와 함께 같이 쓰인다.

- load : 개발자가 작성한 문서를 모두 로딩하면 실행하도록 하는 이벤트 종류
++ onload
예시) window.onload = alert("정신차려라 재기야!"); - 브라우저를 켬 과 동시에 나오게 하는

- resize : 디바이스 영향(태블릿, 모니터, 핸드폰) >  문서화면 크기가 변경되었을 때 사용하는 이벤트 종류

2) 마우스(mouse)이벤트 - 데스크탑을 기반으로 하는

- click , double click : (싱글, 더블) - 버튼이나, 특정요소들을 클릭했을때 실행되도록 하는 이벤트

- mouse over, mouse out : 한 구성이다. 버블링이 일어나는 이벤트
  > 특정요소 위에 커서를 올렸을때 / 떠났을때 실행되도록 하는 이벤트

- mouse enter, leave : 기본 디폴트 값에서 버블링이 일어나지 않는 이벤트
  > 특정요소 위에 커서를 올렸을때 / 떠났을때 실행되도록 하는 이벤트

자바스크립트 안에 함수를 적을 때는 : 특수기호(./-?!) 와 공백을 만들지 못한다.

3) 키보드(Keyboard) 이벤트

- key down : 사용자가 키를 누르고 있는 동안 발생되는 이벤트

- key up : 사용자가 키를 눌렀다가 떼었을때 즉, 행동을 완료했을 때 발생되는 이벤트

4) 폼(form) 이벤트 (내가 채우기)

- change : 이벤트 메서드는 선택한 폼 요소의 value 값을 새 값으로 바꾼다. focus가 다른 요소로 이동하면 이벤트 발생

- focus : 이벤트 메서드는 대상 요소로 포커스가 이동하면 이벤트 발생
focusin() 이벤트 메서드는 대상 요소의 하위 요소 중 입력 요소로 포커스가 이동하면 이벤트 발생
focusout() 이벤트 메서드는 대상 요소의 하위 요소 중 입력 요소에서 외부 요소로 이동하면 이벤트 발생

- blur : 이벤트 메서드는 포커스가 대상 요소에서 다른 요소로 이동하면 이벤트 발생

- submit : 버튼 클릭을 통해 데이터를 전송하기는 하나, 결과적으로 폼의 모든 데이터를 전송하는 것이라, 이벤트를 받는 대상은 form태그입니다.

5. 위치 이벤트

- screenX : 현재 화면을 기준으로 x축으로 부터 얼만큼 이동한 곳인지 확인
- screenY : 현재 화면을 기준으로 y축으로 부터 얼만큼 이동한 곳인지 확인
** 화면의 크기가 커질수록(디바이스 종류에 따라) 내가 가져올수 있는 값이 커지게 된다.

** 타입스크립트 

tsc -v 타입스크립트의 버전을 확인해볼 수 있는 커맨드이다.
오류가 뜬다면 윈도우 파워셀 을 활성화 한다음 관리자 권한으로 실행시킨다.

🔹 해결 방법
✅ 1. TypeScript가 설치되어 있는지 확인
먼저, TypeScript가 설치되었는지 확인하려면 다음 명령어를 실행하세요.

npm list -g typescript
TypeScript가 설치되어 있다면 설치 경로가 표시됩니다.
"empty"가 출력되거나 아무 결과도 나오지 않으면 설치되지 않은 것입니다.

✅ 2. TypeScript 설치
TypeScript가 설치되지 않았다면, 다음 명령어로 전역(global) 설치하세요.!!!!!!

npm install -g typescript (전역설치를 해야한다.)
설치가 완료되면 다시 tsc -v를 실행해보세요.

안된다면 윈도우 자체 권한을 변경해줘야 한다. 그 다음 순서 
get-help Set-ExecutionPolicy 입력 > 'y' 입력 > set-ExecutionPolicy RemoteSigned > 'y' 입력

tsc script.ts 커맨드로 컴파일 할 경우 JS 가 생성될 것이다. (실행은x 해당 파일이 보여진것이다.)

타입스크립트는 파일이 쪼개져 있어도 동일한 변수명이 발견되는 순간 같은 파일로 인식이 되서
오류가 생긴다.  ** 그래서 해당 옵션을 걸어 변수명이 걸리지 않게 동작을 시켜야 한다.

우리가 폴더를 만들게 된다면 (src)폴더안에 JS/TS 파일을 넣어서 보관할수 있다.
tsc src/script.ts 이런식으로 커맨드를 입력하면 된다.

npm i tsx 또는 컴파일을 하지 않고 다이렉트로 실행을 시키는 방법이다. (터미널에 바로 실행 해당파일x)
tsx src/script.ts
Hello World!	이런 식으로 바로 실행되어져서 보여질 것이다.

** 물론 npm i tsx -g tsx 또한 전역설치를 해줘야 한다. tsx -v 를 하면 그러면 정상작동 될 것이다


둘다 (실행과 동시에 해당 파일도 만들고 싶다면) 예약어를 만들면 되는데

package.json 파일로 들어간 뒤 
"scripts": {  >> 해당 태그에서
    "test": "echo \"Error: no test specified\" && exit 1",
    "ts_node": "tsc && tsx src/script.ts" >> 이렇게 예약어를 만들어서 쓰면 된다.

그 후 순서 tsc --init  이 커맨드 입력을 한 뒤 파일이 생성!  >tsconfig.json!
해당 파일 안에서 옵션을 설정해주면 된다. > "include": ["src"] 이렇게 설정을 한 뒤
다시 터미널에서 npm run ts_node 커맨드를 입력하면 실행과 동시에 JS파일이 형성이 된다.
